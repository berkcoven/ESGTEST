How to Customize

The template includes an optional development flow using npm and grunt. If you want to try it you can skip to the section, Development Flow (optional), where we explain what you'll need to install in order for it to work.

Template Files

This is the main version of the template that you can integrate with your desired task runner / compilers / development flow. The files of the template can be found inside the HTML Template directory. This includes the source files (sass, js, etc) that you can use to customize the template, as well as the compiled files (css, js bundle).

Development Flow (optional)

The development flow is based on the use of npm as a package manager and grunt as a task runner.
First you will need to install the following:

  • Node

  • npm

Once you have installed them and verified that they are functioning correctly, navigate to HTML Template - Optional. Here you will find a file called package.json, this file contains the development and production dependencies of the template (including grunt, the task runner that we will be using), this is what is going to be managed by npm.

npm

In order to install this dependencies, navigate to HTML Template - Optional if you haven't already, and run the command npm install in your terminal.
This will take some minutes as it downloads and installs all the dependencies and related packages that they need. When it's done, you'll notice that a folder called node_modules will be created in the same directory, this folder has all the dependencies that were downloaded, make sure that you don't delete it or you'll have to run npm install again.

Now that we have everything setup, we only need to use grunt for development (note that you don't need to run npm install again, unless you add new dependencies or want to update them or npm itself).

grunt

In the same directory (HTML Template - Optional) you will find a file called Gruntfile.js, this file contains the tasks that will be run each time you execute the grunt command on that same directory and when you make changes to the files of the template with the command running. So you only need to execute it once when you want to start customizing the template, and further changes to files in the src/ directory will trigger an automatic update and refresh of the files (livereload).

To start customizing the template, navigate to HTML Template - Optional and run grunt on your terminal. You'll notice that it will do some work and it should automatically open a tab on your browser, if it doesn't, open a new tab on your browser manually and enter localhost:8099 as the URL (that port number must not be in use by other application for it to work). If you need or want to change the port you can do so by editing the Gruntfile.js file in the connect:build section, you will also find the livereload port number there.

When executing the grunt command and/or when updating files on the src/ directory, the task runner will generate a dist/ folder, this is the production version of the template, i.e. the one to deploy. So, for example, if you want to edit styles, find the corresponding file in the src/sass/ directory, or create a new one and edit the styles, grunt will automatically compile the .sass files into one .css file, generate the dist/ folder again with the new compiled styles, and update the browser tabs using livereload, so you can check the changes instantly.

The process is the same for adding or modifying images, HTML or Javascript files, HTML files and images are copied to dist/ preserving the same directory structure, while Javascript files are transpiled using Babel, bundled using Browserify and minified with UglifyJS into one bundle that is included in all the HTML pages. The entry point for the Javascript bundle is the file src/app.js which is used to require all the files that will be put in the bundle, which can be found in the src/js/ directory.

Twitter Widget

In order to display twitter feed the template includes an express based server to fetch the most recent tweets and a javascript file that queries this server for the information and displays it on the page. Before continuing with the following sections, install node if you don't already have it.

Server

The server file is in the root directory and is called server.js. Inside this file you'll find the url of the website you want to allow access to your server, by default localhost:8099 and the twitter endpoint where you'll have to enter your keys. You can also change the default port which the server will listen to.

After customizing the server.js file, you can start it by navigating to the root directory and entering the command node server.js.

The server must be running in order to be able to display tweets on the widgets..

Widget

The file that queries the server and displays the twitter widgets on the template can be found in js/widget/widget.twitter.js. At the bottom of this file you'll find a count constant that you can edit to modify the amount of tweets that you want to request and the url of the server that you'll need to change when deploying on production.

Instagram Widget

In order to display Instagram feed the template makes use of the Instagram API endpoints. You'll need an access token to be able to use this widget, you can find information on how to get your access token here.

Widget

The file that is used to display the instagram widget on the template can be found in js/widget/widget.instagram.js. At the bottom of this file you'll find a count constant inside the createInstagramItems function that you can edit to modify the amount of instagram images that you want to request. Also, below that you'll find the insta_access_token constant where you'll need to enter yours for it to work.

Google Maps

The template includes 3 different custom google map widgets. One for the sidebar, and two other for the body of the template pages. All of them have a configuration file associated that can be found in js/sidebar/sidebar.map.js and js/body/body.map.js respectively. You can edit those files to input your desired location and marker.

Google API Key

For the map to work, you need to get an API Key from Google here and replace it on the script tags where it says key=YOUR_API_KEY. The callback argument must point to a function that will be used to create the map.

You will find those script tags on the HTML pages that have a map widget.

Introduction

The elements library is a template element previewer and replicator. It lets you preview and easily copy the most important elements of the template to help customization. You access it by opening the index.html file inside the folder CSS and JS Elements Library.

How to Use

First, open "CSS and JS Elements Library"/index.html to access the elements library. You'll see all the available categories filtered by type (CSS / JS)

Preview

To preview elements, just click on a category. Inside you'll find all the elements associated to that category.

Copy

Instead of having to search through the HTML code to select and copy elements, we provide a simple way to copy them from the previewer. Just put the mouse cursor over the element that you want to copy and left click it. A tooltip will appear indicating that the element's code has been copied succesfully.

Please note that not all elements are supported, you can only copy elements contained in the following categories and those that don't have a plugin that modifies styles inside (like a slider):

Supported Categories

  • Layout

  • Grid

  • SVG

  • Title

  • Button

  • Form

  • Decoration

  • Banner

  • Table

  • Widget

  • Product Preview

  • Post Preview
